SOP Workflow Steps 1. Request / Message Received ● Customer or team member sends a support message through the platform. ● The system records the request and assigns it to the support queue. 2. Ticket Assignment ● The system or team lead assigns the request to an available Text Support Agent . ● The agent reviews the message and related information. 3. Issue Analysis ● The agent carefully reads the request. ● Identify the problem type (technical issue, account issue, general question, etc.). 4. Solution Identification ● The agent searches the knowledge base or SOP documents . ● Determine the correct solution or response. 5. Response to User ● The agent sends a clear and professional message. ● Provide step-by-step help if required. 6. Escalation (If Needed) ● If the issue cannot be solved, escalate it to: ○ Technical team ○ Project manager

○ Senior support staff 7. Resolution ● Confirm that the issue is solved. ● Update the ticket status to Resolved . 8. Documentation ● Record the solution in the system. ● Update knowledge base if the issue is new. Text Support Life Cycle You can paste this image/diagram in the SOP file. Customer Request │ ▼ Ticket Created │ ▼ Ticket Assigned to Agent │ ▼ Issue Analysis │ ▼ Find Solution │ ▼ Respond to Customer │ ▼ Resolved / Escalated │ ▼ Ticket Closed & Documented
